### [2] 컴퓨터 기본구조
** (1) HW(물리적 존재)**
* ((1)) CPU -> (쓰기) -> Memory / Memory -> (읽기) -> CPU
* ((2)) HW-CPU (연산, 제어)
* ((3)) HW-Memory
    * (((1))) **클락(Clock)** : CPU가 일을 진행하는 템포(박자) / **워드(Word)** : CPU가 한번에 처리할 수 있는 단위

** (2) SW(가상 존재) (응용프로그램 <- 유틸리티 <- O/S <- H/W)**
* ((1)) 메모리(주소, 값 저장 = 1byte(값(데이터)을 저장하는 최소단위)) : 값을 저장하는 공간이 연속적으로 구성
    * (((1))) 32bit 컴퓨터 = 4Gbyte (10bit = KB, 20bit = MB, 30bit = GB, 40bit = TB, 50bit = PB...)
    * (((2))) 사람이 읽고 간결하게 쓰기 위해 2진수를 8, 16진수로 표현법으로 나타냄
        * -> (16진수 표현법)(0x00~0xff) / (8진수 표현법)(0000~7777)
    * (((3))) 메모리는 지우지 않고 덮어쓴다 (이전 데이터를 볼수x)
* ((2)) 컴퓨터에서 정보가 들어오면 입력, 나가면 출력
* ((3))* **페이지 기법(Paging)** : 가상기억장치(프로그램)와 물리 메모리를 고정된 크기(Page/Frame)로 나누어, 불연속적으로 적재하여 관리하는 운영체제 메모리 관리 방식
* ((4))* **메모리 인터럽트(Memory Interrupt)** : 시스템 메모리 접근 오류(예: 잘못된 주소 접근, 페이지 폴트) 발생 시 CPU가 하던 작업을 멈추고 예외 처리 루틴(ISR)을 수행해 문제를 해결하는 메커니즘
* ((5)) **버퍼 메모리(Buffer Memory)** : 속도 차이가 나는 두 장치(예: CPU와 하드디스크) 사이에서 데이터를 주고받을 때, 데이터 유실을 막고 효율을 높이기 위해 일시적으로 데이터를 저장하는 임시 공간
